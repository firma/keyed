流程

  • master 用于线上发布
  • feature-xxx 用于添加功能，添加完删除
  • develop 用于合并feature分支。
  • release 用于发布前的测试，发布完删除
  • bugfix-xxx 用于修复develop和master分支的bug。
  • 主要流程有三个：添加新功能，修复Bug和发布新版本。
